Visualizer

A draggable on-screen window backed by an EditableImage, for eyeballing noise fields, curves, and point sets while you tune them. Rendering runs through a single reused pixel buffer, so re-rendering every frame is cheap. 

local viz = Atlas.Visualizer.new({ width = 256, height = 256, title = "fBm" })
viz:render(function(x, y)
	local v = simplex:sample2D(x * 0.02, y * 0.02) * 0.5 + 0.5
	return v, v, v
end)

interface VisualizerOptions {
width: number?--

Image width in pixels, at most 1024. Defaults to 256.

height: number?--

Image height in pixels, at most 1024. Defaults to 256.

parent: Instance?--

Where to parent the window. Defaults to the local player's PlayerGui.

title: string?--

Window title. Defaults to "Atlas".

pixelScale: number?--

On-screen magnification of the image. Defaults to 1.

}

Creates a visualizer window. On the server (or in other headless contexts) options.parent must be provided, since there is no local PlayerGui.

render​

</>
Visualizer:render(callback: PixelCallback) → ()

Renders the image by calling callback once per pixel. Returned channels are clamped to [0, 1].

draw​

</>
Visualizer:draw(callback: DrawCallback) → ()

Hands the raw RGBA buffer to callback for direct writes, then uploads it. Faster than Visualizer:render when the callback can batch its own work, and the natural fit for Presets.curve and Presets.points.

destroy​

</>
Visualizer:destroy() → ()

Closes the window and releases the EditableImage. Safe to call twice.
